# 遊戲設計文件 — CyberDesk 資安AI考核遊戲

## 遊戲概念

一款模擬 Windows 11 桌面的互動式考核遊戲，用於測驗員工的**資訊安全意識**與**生成式 AI 正確使用觀念**。

玩家在一個假的 Win11 桌面環境中，透過操作桌面上的應用程式（郵件、瀏覽器、檔案總管、設定等）來完成各關卡的挑戰。

## 目標對象

- 一般企業員工（非技術背景）
- 難度定位：日常工作場景，不需要專業技術知識
- 目的：資安意識培訓 + AI 使用規範教育

## 核心玩法循環

```
關卡開始
  ↓
顯示謎題提示（情境描述 + 任務目標）
  ↓
玩家在 Win11 桌面上自由探索
  ↓
透過操作桌面應用完成任務
  ↓
系統判定是否過關
  ↓
顯示結果 + 教學解說
  ↓
進入下一關
```

## 過關機制

每關採用以下兩種機制之一（或混合）：

### 1. 找到正確答案
- 玩家需要在桌面環境中找到特定線索並做出正確選擇
- 例：從多封郵件中找出釣魚信並點擊「回報」按鈕
- 適用於：辨識型題目（釣魚信、AI生成內容、可疑檔案）

### 2. 完成操作流程
- 玩家需要依序完成一系列正確操作步驟
- 例：發現可疑檔案 → 不要打開 → 右鍵回報 → 通知IT部門
- 適用於：流程型題目（密碼修改、事件處理、AI工具使用）

## 關卡主題分佈

| 類別 | 關卡數 | 佔比 |
|------|--------|------|
| 資訊安全 | 6 關 | 50% |
| 生成式 AI | 6 關 | 50% |
| **合計** | **12 關** | **100%** |

詳細關卡設計見 [levels.md](levels.md)。

## 遊戲流程

```
主選單
  ├─ 開始考核（從第1關開始）
  ├─ 選擇關卡（可跳關，或鎖定需循序解鎖）
  └─ 成績總覽（各關得分與總分）

每關流程：
  1. 關卡載入畫面（顯示關卡編號與主題）
  2. 情境說明面板（背景故事 + 任務目標）
  3. 桌面互動階段（玩家自由操作）
  4. 過關判定
  5. 結果面板（正確/錯誤 + 教學解說）
  6. 進入下一關 or 返回選單
```

## 計分系統

- 每關滿分 100 分
- 一次答對：100 分
- 嘗試多次後答對：部分分數（例如 60 分）
- 答錯但看完解說：30 分（至少學到了）
- 總分 = 各關分數加總
- 及格標準：總分 70% 以上

## UI/UX 設計原則

1. **擬真度**：桌面盡量仿真 Win11，降低學習門檻
2. **引導性**：每關開始有明確提示，不讓玩家迷失
3. **即時回饋**：操作正確/錯誤時有視覺提示
4. **教育性**：過關後一定要有解說，告訴玩家「為什麼」
5. **無壓力**：允許重試，重點是學習而非懲罰
