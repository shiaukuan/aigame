# CyberDesk 遊戲改進建議報告

> 基於完整 codebase 審查（15 個關卡 + desktop.gd + 架構），以下為分類建議。

---

## 一、遊戲體驗（UX）

### 1.1 玩家迷路問題
- **現狀**：Level 8-15 的 `setup_desktop()` 為空，玩家完全沒有視覺引導，不知道要打開哪個 App
- **建議**：為每關加入「首次提示 toast」——關卡開始 2 秒後在畫面下方淡入一句話（如「試試看設定裡的應用程式頁面」），5 秒後淡出。不直接說答案，但給方向感

### 1.2 關卡結果頁缺乏回顧
- **現狀**：過關後只顯示教學重點 (teaching_points)，但不告訴玩家「你哪裡做錯了」
- **建議**：result 畫面增加一區「你的操作回顧」，列出 `GameState.wrong_actions` 並用紅字標注（例：「你點擊了釣魚連結 ✗」「你上傳了機密檔案 ✗」），讓玩家知道自己扣分的原因

### 1.3 沒有漸進式提示
- **現狀**：失敗 3 次才出現「查看解答」，中間沒有任何額外線索
- **建議**：失敗 1 次後給一句模糊提示，失敗 2 次後給更具體的提示（可從 `task_hint` 拆成多段），失敗 3 次才給解答。降低挫敗感

### 1.4 視窗系統限制
- **現狀**：同時只能開一個視窗，打開新 App 會關閉舊的
- **建議**：允許同時開 2 個視窗（並排或可切換），這對 Level 11（需要在 AI 助手和記事本間切換）、Level 13（需要在程式碼編輯器的不同 tab 切換）特別有幫助

---

## 二、難度與教學設計

### 2.1 前幾關太簡單，後面斷崖
- **現狀**：Level 1-4 有明確的視覺引導（taskbar 閃爍、highlight_app），Level 8 開始完全沒有引導
- **建議**：做一條平滑的難度曲線——Level 5-7 減少一點引導但保留提示，Level 8-11 給「首次提示 toast」，Level 12-15 才完全無引導

### 2.2 Level 7 資料分級 badge 顏色太直覺
- **現狀**：綠色 = 可上傳、紅色 = 不可上傳，玩家不用讀文字就能猜對
- **建議**：所有分級 badge 用同一個中性色（灰藍），迫使玩家閱讀分級名稱和描述來判斷

### 2.3 Level 12 對非技術人員太難
- **現狀**：需要看懂 Python 程式碼的 SQL injection、明文密碼、硬編碼金鑰
- **建議**：game-design.md 定位是「一般企業員工（非技術背景）」。可以在每段程式碼旁加一個「📖 白話說明」按鈕，點擊後用非技術語言解釋這段程式碼在做什麼

### 2.4 缺少「為什麼」的教學
- **現狀**：過關後的 teaching_points 告訴你「應該怎麼做」，但沒有解釋後果
- **建議**：加入真實案例——例如 Level 1 結束後提到「2023 年某公司因員工點擊釣魚信導致 XX 萬筆資料外洩」，增加記憶點

---

## 三、關卡設計細節

### 3.1 Level 1（釣魚信）
- 「[外部郵件]」標籤已經是明顯線索，釣魚信太容易辨認
- **建議**：加入 1-2 封有「[外部郵件]」標籤但是正常的信（例如合作廠商通知），讓玩家不能只靠標籤判斷

### 3.2 Level 4（可疑檔案）
- 右鍵選單的「掃描病毒」功能直接告訴你檔案是否危險
- **建議**：掃描結果改為更模糊的描述（「此檔案的副檔名與內容類型不符」），不直接說「偵測到威脅」

### 3.3 Level 6（USB）
- 目前只要「安全退出」就過關，即使打開了所有檔案
- **建議**：如果打開了 USB 檔案，即使退出也只能拿 60 分（目前已實作），但應在結果頁明確告知「你打開了 X 個檔案，真實情況中可能已中毒」

### 3.4 Level 9（AI 幻覺）
- 段落很長，在 640x408 的視窗內閱讀體驗差
- **建議**：加入可展開/收合的段落，或允許點擊段落進入全文閱讀模式

### 3.5 Level 10（Prompt Injection）
- 兩個 tab（對話記錄 + 防護設定）的操作較複雜
- **建議**：加入 tab 上的未完成提示（例如對話記錄 tab 上顯示「⚠️ 2」表示有 2 筆攻擊尚未標記）

### 3.6 Level 11（AI 使用流程）
- 多步驟流程正確順序太嚴格（必須先讀規範再讀需求再送 AI 再編輯再儲存）
- **建議**：允許部分步驟順序彈性（如先讀需求再讀規範也可以），但「送機密資料給 AI」仍然是錯的

## 四、技術改進

### 4.1 desktop.gd 過大
- **現狀**：超過 2800 行，包含所有 App 的 UI 建構、視窗系統、開始選單、時鐘等
- **建議**：將各 App 的預設內容（email、settings、browser 等）拆分到獨立的 `.gd` 檔案，desktop.gd 只保留框架和委派邏輯

### 4.2 沒有存檔機制
- **現狀**：重新整理瀏覽器或關閉 App 就會失去所有進度
- **建議**：用 `localStorage`（Web 版）或 `FileAccess`（桌面版）儲存 ScoreManager 的 scores/attempts，讓玩家可以中途離開再回來

### 4.3 Feedback box 重疊問題
- **現狀**：feedback box 位置是硬編碼的，有些關卡的 feedback box 會蓋住「完成作答」或「查看解答」按鈕
- **建議**：統一 feedback box 的位置策略——總是在視窗頂部或底部固定位置，不被其他元素遮擋

### 4.4 缺少載入/過場動畫
- **現狀**：關卡之間的切換是瞬間的
- **建議**：加入簡單的淡入淡出（0.3 秒），讓轉場更流暢

---

## 五、可及性（Accessibility）

### 5.1 色覺障礙
- 遊戲大量依賴紅/綠色區分安全/危險
- **建議**：除了顏色外，加入圖示區分（✓ / ✗ 或 🟢 / 🔴 emoji），確保不靠顏色也能判斷

### 5.2 文字大小
- 大部分 UI 文字是 11-12px，在 1280x720 的遊戲畫面上偏小
- **建議**：考慮提供「大字模式」，將所有 font_size 放大 2px

### 5.3 沒有新手引導
- 第一次打開遊戲直接進入第一關，玩家不知道「關卡提示.docx」的存在
- **建議**：第一關開始前加入 30 秒的互動教學——「這是你的桌面，右上角的📝是關卡提示，每關開始請先看它」

---

## 六、內容擴充方向

### 6.1 社交工程場景不足
- 目前只有 Level 1（釣魚信）涉及社交工程
- **建議**：新增「電話社交工程」關卡（假冒 IT 打電話要密碼）、「物理社交工程」關卡（尾隨進入辦公室）

### 6.2 缺少事件回應 (Incident Response) 關卡
- 所有關卡都是「預防」導向
- **建議**：新增「你的帳號已被盜」「你發現同事在用禁用 AI」等事件回應場景，教玩家該通報誰、怎麼處理

### 6.3 多人/競賽模式
- **建議**：加入排行榜功能，公司內部可比較分數，增加參與動機

---

| P1 | 1.1 首次提示 toast | 中後段關卡探索難度太高 |
| P1 | 1.2 操作回顧 | 玩家不知道自己錯在哪 |

| P2 | 1.3 漸進式提示 | 降低挫敗感 |

| P2 | 4.4 過場動畫 | 體驗打磨 |
